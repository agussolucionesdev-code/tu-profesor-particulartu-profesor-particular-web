# Checklist de aceptación visual y funcional

## Fidelidad visual
- [ ] El logo aparece correctamente en navbar, drawer y footer.
- [ ] Las fotos reales cargan sin rutas rotas.
- [ ] El hero mantiene composición, jerarquía, badges y CTAs.
- [ ] La paleta azul/verde/blanco cálido coincide con la referencia.
- [ ] Las tipografías se ven como la referencia.
- [ ] Las sombras, bordes y radios no fueron simplificados en exceso.
- [ ] El diseño mobile conserva navegación, jerarquía y legibilidad.
- [ ] No aparece estética genérica tipo plantilla.

## Interacciones
- [ ] Progress bar de scroll funciona.
- [ ] Navbar cambia en scroll.
- [ ] Links hacen scroll suave.
- [ ] Sección activa se marca correctamente.
- [ ] Drawer mobile abre/cierra con backdrop y Escape.
- [ ] Palabras rotativas del hero funcionan.
- [ ] Reveal on scroll funciona.
- [ ] Contadores animados funcionan.
- [ ] Selector nivel/materia muestra mensajes correctos.
- [ ] Link WhatsApp del selector cambia según selección.
- [ ] FAQ abre/cierra correctamente.
- [ ] Tabs de Sobre mí funcionan.
- [ ] Selector de modalidades funciona.
- [ ] Cards con tilt funcionan en desktop sin romper mobile.

## Técnica
- [ ] `npm install` funciona en frontend.
- [ ] `npm run dev` funciona en frontend.
- [ ] `npm run build` funciona en frontend.
- [ ] No hay errores en consola.
- [ ] No hay assets faltantes.
- [ ] Variables de entorno documentadas.
- [ ] Backend responde `GET /health`.
- [ ] CORS configurado por ambiente.
- [ ] El deploy de Vercel toma `frontend/` como root.
- [ ] El deploy de Render toma `backend/` como root.
